# Introduction

Get started with TSTyche. The example below shows how a type testing can help to debug a utility type.

## Prerequisites

Make sure you're using Node.js LTS version and internet connection is available. To learn more, see the [System Requirements](../reference/requirements.mdx) page.

## Install TSTyche

Use your favorite package manager to add `tstyche` (and optionally `typescript`) to a project:

<Tabs items={["npm", "yarn", "pnpm"]} storageKey={"preferredPackageManager"}>
<Tabs.Tab>

```shell
npm add -D tstyche typescript
```

</Tabs.Tab>
<Tabs.Tab>

```shell
yarn add -D tstyche typescript
```

</Tabs.Tab>
<Tabs.Tab>

```shell
pnpm add -D tstyche typescript
```

</Tabs.Tab>

It is recommended to create a [`tsconfig.json`](https://typescriptlang.org/docs/handbook/tsconfig-json.html) file as well. Run `tsc --init` and TypeScript will generate it.

<Note>

The following steps assume that you installed the `typescript` package and added the `tsconfig.json` file. TSTyche works just fine without a TSConfig file and TypeScript installed. To learn more see the [JavaScript Project](./javascript-project.mdx) page.

</Note>

## Define a Type

Create a `MethodLikeKeys.ts` file with the following content:

```ts filename="./MethodLikeKeys.ts"
type MethodLike = (...args: any) => any;

export type MethodLikeKeys<T> = keyof {
  [K in keyof T as T[K] extends MethodLike ? K : never]: T[K];
};
```

The `MethodLikeKeys` utility should construct a union type by picking up names of method like keys of the `T` type.

## Add a Type Test

Create a `MethodLikeKeys.tst.ts` file in the same directory:

```ts filename="./MethodLikeKeys.tst.ts"
import { expect } from "tstyche";
import { MethodLikeKeys } from "./MethodLikeKeys.js";

interface Sample {
  description: string;
  getLength: () => number;
  getWidth?: () => number;
}

expect<MethodLikeKeys<Sample>>().type.toEqual<"getLength" | "getWidth">();
```

Here the `expect()` function defines an easy to read assertion. There are two method like keys in the `Sample` interface, hence the resulting type must equal `"getLength" | "getWidth"`.

<Tip>

By convention, it is recommended to suffix the functional tests with `.test.*` and the type tests with `.tst.*`. To learn more, see the [TypeScript Project](./typescript-project.mdx) page.

</Tip>

## Iterate

The `getWidth` key is marked as optional and was not picked up by the `MethodLikeKeys` type. Here is the fixed type:

```ts /Required<T>/
export type MethodLikeKeys<T> = keyof {
  [K in keyof T as Required<T>[K] extends MethodLike ? K : never]: T[K];
};
```

Run the test again and it will pass:

```ansi
[34muses[0m TypeScript 5.2.2[90m with ./tsconfig.json[0m

[32mpass[0m [90m./[0mMethodLikeKeys.tst.ts
```
