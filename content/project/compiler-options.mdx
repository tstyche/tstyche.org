# Compiler Options

Compiler options for type testing are loaded from the nearest `tsconfig.json`. If none is found, TSTyche falls back to a set of defaults.

## TSConfig File

TSTyche searches for the nearest `tsconfig.json` that includes the test file. The path of the discovered TSConfig file is included in the report:

```ansi
[34muses[0m TypeScript 5.9.3[90m with ./typetests/tsconfig.json[0m
```

If a particular test file is not included in any of the TSConfig files found in enclosing directories, TSTyche falls back to [default compiler options](#default-compiler-options). The report looks like this:

```ansi
[34muses[0m TypeScript 5.9.3[90m with baseline TSConfig[0m
```

To specify the path to `tsconfig.json` explicitly, pass the path with the [`--tsconfig`](../reference/command-line.mdx#tsconfig) command line option or use the [`tsconfig`](../reference/config-file.mdx#tsconfig) configuration option.

<Tip>

To inspect the resolved compiler configuration, pass the path of the TSConfig file to `tsc` together with the `--showConfig` option:

```shell
tsc --project ./typetests/tsconfig.json --showConfig
```

</Tip>

## Default Compiler Options

TSTyche sets the following default compiler options:

```json
{
  "allowJs": true,
  "checkJs": true,
  "allowImportingTsExtensions": true,
  "exactOptionalPropertyTypes": true,
  "jsx": "preserve",
  "module": "nodenext",
  "moduleResolution": "nodenext",
  "noUncheckedIndexedAccess": true,
  "noUncheckedSideEffectImports": true, // only for TypeScript >=5.6
  "resolveJsonModule": true,
  "strict": true,
  "target": "esnext"
}
```

<Note>

Your code editor may use different default compiler options. This is why TSTyche and your editor may see types differently. If that is an issue, add a `tsconfig.json` to your project — both TSTyche and your editor will use it.

</Note>

## Language Service Plugins

To work with files like `.svelte` or `.vue`, TypeScript relies on [language service plugins](https://github.com/microsoft/TypeScript/wiki/Writing-a-Language-Service-Plugin). Code editors load them via extensions.

TSTyche makes sure the plugins are loaded, but they must be installed and explicitly listed in a TSConfig file. For example, if your type tests import from `.svelte` files, install the `typescript-svelte-plugin` package:

<Tabs items={["npm", "yarn", "pnpm"]} storageKey={"preferredPackageManager"}>
<Tabs.Tab>

```shell
npm add -D typescript-svelte-plugin
```

</Tabs.Tab>
<Tabs.Tab>

```shell
yarn add -D typescript-svelte-plugin
```

</Tabs.Tab>
<Tabs.Tab>

```shell
pnpm add -D typescript-svelte-plugin
```

</Tabs.Tab>
</Tabs>

Add it to the language service [`plugins`](https://typescriptlang.org/tsconfig#plugins) list in `tsconfig.json`:

```json
{
  "compilerOptions": {
    "plugins": [{ "name": "typescript-svelte-plugin" }]
  }
}
```
